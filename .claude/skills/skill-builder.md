# Skill: Skill 构建器 (Skill Builder)

## 触发条件
当用户提到以下意图时激活：

- "把这个打包成 skill"、"把刚才做的变成 skill"
- "生成一个 skill"、"create a skill from this"
- "我想让别人也能用这个流程"
- "把这个能力固化下来"、"打包 skill"

## 概述

从当前对话中提炼可复用的 Skill，生成标准的 `.claude/skills/*.md` 文件 + 配套代码，并完成打包（安全检查、README、install.sh、.gitignore）。

**本 skill 只负责构建和打包，不负责推送到 GitHub**——推送由 `skill-publisher.md` 单独完成。构建完成后会询问用户是否需要继续发布。

---

## 第 1 步：分析对话，提炼流程

**回溯整个对话历史**，提取：

1. **用户完成了什么任务**：目标是什么、解决了什么问题
2. **工作流程**：分几步完成、每步做了什么
3. **使用了哪些工具**：Python 脚本、bash 命令、MCP 工具、WebSearch、WebFetch 等
4. **关键决策点**：哪里需要用户确认、哪里有分支逻辑
5. **输入和输出**：接收什么输入、产出什么结果
6. **代码产物**：对话中创建或修改了哪些代码文件

总结后展示给用户：

```
📋 任务分析
━━━━━━━━━━━
任务：[一句话描述]
场景：[适用于什么场景/谁会用]
步骤：
  1. [步骤名] — [做了什么]
  2. [步骤名] — [做了什么]
  ...
工具/脚本：
  - [文件路径] — [用途]
用户交互点：
  - [哪里需要用户确认/输入]
```

> "根据我们刚才的对话，我提炼出了以上流程。这个理解准确吗？有没有需要调整的地方？"

## 第 2 步：设计 Skill 架构

根据任务复杂度，决定 skill 结构：

| 复杂度 | 建议 |
|--------|------|
| 简单（≤3 步） | 1 个 skill 文件 |
| 中等（4-6 步） | 1 个主 skill，步骤写在同一个文件里 |
| 复杂（>6 步或有独立子流程） | 1 个编排 skill + 多个子 skill |

向用户说明架构决策并等待确认。

## 第 3 步：生成 Skill 文件

按照标准模板生成 `.claude/skills/*.md`：

```markdown
# Skill: [中文名] ([English Name])

## 触发条件
当用户提到以下意图时激活：
- "[关键词1]"、"[关键词2]"

## 概述
[一段话说明做什么、适用场景]

## 前置输入
开始前必须已知：
1. [需要什么输入]

## 工作流程

### 第 1 步：[步骤名]
[具体说明]
**交互节点**：[需要用户确认什么]

### 第 2 步：[步骤名]
...

## 错误处理
[常见错误和应对]

## 输出格式
[最终产出]
```

**生成原则**：
1. **触发条件要自然**：用用户日常会说的话，中英文都覆盖
2. **流程写清楚但不写死**：描述"做什么"而非嵌入代码
3. **保留交互节点**：关键步骤必须有用户确认
4. **代码放 `scripts/`**：skill 里只写调用方式，不嵌入大段代码
5. **通用化**：去掉特定案例细节（公司名、产品名等），用通用描述
6. **独立可用**：既能被其他 skill 调用，也能直接触发

## 第 4 步：处理配套代码

检查对话中是否创建了代码文件：

**有代码产物**：
1. 确认文件已保存在 `scripts/` 目录
2. 去除硬编码的个人信息（邮箱、路径、密码等）
3. 确保代码通用，去除特定业务逻辑
4. 生成或更新 `requirements.txt`

**无代码产物**：
- 仅生成 `.md` skill 文件即可

## 第 5 步：用户审核

完整展示生成的 skill 文件：

> "以下是生成的 Skill 文件，请审核：
>
> [完整 .md 内容]
>
> 你可以：
> 1. ✅ 确认，保存到 `.claude/skills/`
> 2. ✏️ 告诉我修改意见
> 3. 🔄 重新生成"

确认后保存到 `.claude/skills/` 目录。

## 第 6 步：安全检查（强制）

扫描所有待打包文件，检查敏感信息：

- 邮箱密码 / App Password
- API Key（`SG.` / `re_` / `sk_` / `ghp_` 开头）
- 硬编码邮箱地址
- 绝对路径（`/Users/xxx/...`、`/home/xxx/...`）
- IP 地址
- token / secret / credential 赋值

发现问题时提示并帮用户清理。

## 第 7 步：生成项目文件

生成或更新以下文件：

1. **README.md**：项目名、功能概览、安装方法（一键 + 手动）、使用方法、项目结构
2. **install.sh**：自动发现并复制 skills + commands + scripts，安装 Python 依赖
3. **.gitignore**：排除 .DS_Store、__pycache__、.env、.claude/settings.local.json、临时数据

## 第 8 步：生成斜杠命令

如果 skill 适合通过斜杠命令触发，在 `.claude/commands/` 中生成对应的命令文件：

```markdown
---
name: [命令名]
description: [一句话描述]
---

[触发 skill 的指令]
```

## 第 9 步：构建完成

> "✅ Skill 构建完成！
>
> 📁 生成的文件：
> - `.claude/skills/xxx.md`
> - `.claude/commands/xxx.md`
> - `README.md`
> - `install.sh`
> - `.gitignore`
>
> 你可以在新对话中输入 `/xxx` 测试。
>
> **需要发布到 GitHub 吗？** 说 \"发布\" 或 `/skill_publisher` 即可。"

---

## 独立使用场景

### 只做安全检查
```
用户："检查一下有没有敏感信息"
→ 只执行第 6 步
```

### 只生成 README
```
用户："帮我写个 README"
→ 只执行第 7 步中的 README 部分
```

## 注意事项

- 生成的 Skill 必须通用化——去除对话中的特定案例细节
- 安全检查是强制步骤，不能跳过
- 参考项目中已有的 skill 文件作为格式范例
- 构建完成后提示用户可以继续发布，但不自动发布
